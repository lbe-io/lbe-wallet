# i18n 指南

更新时间：2026-05-14  
语言：简体中文

## 目录结构
- `src/locales/resources.ts`：运行时 locale 注册表、支持语言列表、共享 key 类型。
- `src/locales/index.ts`：locale helper 的统一导出入口。
- `src/locales/en/translations.json`：运行时 key 集合的基准来源。
- `src/locales/zh-CN/translations.json`：简体中文翻译。
- `src/locales/zh-TW/translations.json`：繁体中文翻译。
- `src/locales/ja/translations.json`：日文翻译。
- `src/locales/zh/translations.json`：历史兼容文件，运行时不加载。
- `src/i18n/*`：i18n 初始化、语言检测、单例绑定。
- `src/popup/hooks/useTranslation.ts`：业务层推荐使用的翻译 hook。

## 入口规则
- 业务 UI 统一使用 `src/popup/hooks/useTranslation.ts` 中的 `useTranslation()`。
- 功能组件内不要直接从 `react-i18next` 导入 `useTranslation`。
- `react-i18next` 只保留在 i18n bootstrap 层使用。

## Key 命名规则
- 使用小写、点分隔语义 key。
- 允许根前缀：`common.*`、`page.*`、`component.*`、`error.*`、`error.api.*`、`validation.*`。
- key 需表达业务语义，不要直接用英文文案本身。
- 禁止 `text1`、`title2`、`btnText` 这类占位式命名。
- 不要因文案相似而复用不同业务场景 key。
- 共享文案放 `common.*`，页面文案放 `page.*`，组件文案放 `component.*`，校验文案放 `validation.*`，接口/运行时失败放 `error.api.*`。

## 新增文案流程
1. 先在 `en` 中新增语义 key。
2. 同步在 `zh-CN`、`zh-TW`、`ja` 添加同名 key。
3. 在业务中通过 `@/popup/hooks/useTranslation` 的 `t()` 使用。
4. 如删除或重命名 key，需同 PR 更新所有语言文件。
5. 执行 `corepack yarn compile`。
6. 执行 `corepack yarn build`。
7. 如需 i18n 范围 lint，执行 `corepack yarn exec eslint src`。
8. 回扫页面是否残留硬编码文本。

## 禁止项
- 不改业务逻辑。
- 非必要不改样式（仅在文案长度导致布局问题时调整）。
- 不新增 i18n 库（除非明确说明现有方案不足）。
- 不删除现有运行时语言。
- 不允许多语言 key 漂移。
- 不以 `any` 处理翻译类型链路。
- 不保留 `???`、`????` 这类占位值。

## 示例 key
- `common.confirm`
- `common.cancel`
- `page.login.title`
- `page.login.password.placeholder`
- `component.header.language.select`
- `error.api.network`
- `validation.wallet.password.required`

## 排查硬编码文案
- 优先扫描 `src/popup`、`src/components`、`src/pages`、`src/app`。
- 关注标题、按钮、标签、placeholder、toast、modal、empty/loading、tooltip、aria 文案。
- 跳过调试日志、变量名、非 UI 字面量。
- 检查 `t('...')` 是否仍引用旧 key。
- 快速扫描命令：

```bash
rg -n '"[A-Za-z][^"]+"' src/popup src/components src/pages src/app --glob '!**/translations.json'
```

## Key 同步检查
- `en`、`zh-CN`、`zh-TW`、`ja` 应保持一致 key 集合。
- 可使用 `src/locales/validator.ts` 做覆盖率校验。
- 快速 parity 命令：

```bash
node -e "const fs=require('fs'); const l=['en','zh-CN','zh-TW','ja']; const d=Object.fromEntries(l.map(x=>[x,JSON.parse(fs.readFileSync('src/locales/'+x+'/translations.json','utf8'))])); const ref=Object.keys(d.en).sort(); for(const x of l){ const keys=Object.keys(d[x]).sort(); console.log(x, ref.length===keys.length && ref.every(k=>keys.includes(k)) ? 'ok' : 'mismatch'); }"
```

- 每轮文案变更后，建议至少执行 `corepack yarn compile` 与 `corepack yarn build`。
