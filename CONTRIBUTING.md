# Contributing

## Scope

This repository is published for security transparency and third-party auditability of the wallet codebase.

At this stage, we do not accept external feature contributions or refactor pull requests.

## What We Accept

1. Security vulnerability reports through private channels only.
2. Clarification issues about reproducible build steps and audit documentation.
3. Critical correctness reports with clear reproduction steps.

## What We Do Not Accept

1. Feature pull requests from external contributors.
2. Architectural rewrites or style-only repository-wide changes.
3. Public disclosure of unpatched security vulnerabilities.

## Security Reporting

Do not open a public issue for security problems.

Please follow [SECURITY.md](./SECURITY.md) and use the private reporting channels listed there.

## Pull Requests

Maintainers may close external pull requests that are outside this repository's audit-focused scope.

If your change is security-related and urgent, open a private report first and coordinate with maintainers.
